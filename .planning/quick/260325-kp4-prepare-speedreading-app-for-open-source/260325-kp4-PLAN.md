Replace the minimal existing README with a complete one. The current README is 16 lines and only has a quick start. Write a new README covering:

1. **What it is** — RSVP (Rapid Serial Visual Presentation) speed reading app. Words flash at chosen WPM with an anchor-letter focus point. Reads plain text, EPUB, PDF, or pasted text.

2. **Platform support** — iOS 16+, Android API 26+, Desktop (Linux/macOS/Windows via iced)

3. **Architecture overview** — Shared Rust core (speedreading-app_core) with UniFFI bindings. Unidirectional data flow (Elm/TEA architecture). Platform layers (Swift/Kotlin/iced) contain no business logic. Built with the RMP (Rust Multiplatform) pattern.

4. **Prerequisites** — Nix (provides all tooling via flake.nix), or manually: Rust toolchain, just, rmp CLI. For iOS: macOS + Xcode. For Android: Android SDK (set ANDROID_HOME or android/local.properties).

5. **Build steps** — Use `nix develop` to enter the dev shell, then `just` commands:

   Quick start (after entering nix shell):
     just doctor        # verify prerequisites
     just bindings      # generate FFI bindings for all platforms

   Desktop:
     just run-iced      # build and run desktop app

   Android:
     just android-full  # full pipeline: bindings + cross-compile + assemble APK
     just run-android   # run on connected device/emulator (via rmp)

   iOS (macOS only):
     just ios-full      # full pipeline: bindings + cross-compile + xcframework + build
     just run-ios       # run on simulator (via rmp)

   Individual steps are also available — run `just` to list all recipes.

6. **Optional: direnv** — `.envrc` is included, `direnv allow` enters the nix shell automatically.

Keep tone concise and technical. No badges, no screenshots section, no contributing guide. This is an early-stage personal project.
